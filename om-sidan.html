stacken

Så lite som möjligt

Inget ramverk, ingen bundler, inga beroenden att hålla uppdaterade. Tre HTML-sidor, en stylesheet uppdelad i en delad bas och lite per sida, och en JS-fil för de små interaktionerna — tema, typewriter, live-status. Vill man köra den lokalt räcker en enkel filserver.

Poängen är att sidan ska gå att förstå på fem minuter och fungera i decennier. Ju mindre som kan gå sönder, desto mindre behöver jag laga.

deploy

Från push till live utan att jag rör servern

Förr laddade jag upp filer för hand. Det blev fel förr eller senare — en glömd fil, en bruten länk, fel version. Nu gör GitHub Actions jobbet: jag pushar, en robot kontrollerar och kopierar. Rör jag inte servern kan jag inte råka trasa den.

github actions — deploy.yml 
$ git push origin main

  ├─ ci       node --check · check-links.mjs
  ├─ deploy   rsync -az --delete  ──►  VPS
  └─ live     caddy serverar · https

# hemligheter (nyckel, host, sökväg) ligger i GitHub Secrets — aldrig i repot

pipeline.md

Fyra steg, noll manuella filöverföringar:

  1. Jag pushar till main eller kör den manuellt
  2. CI validerar länkar, JSON och JS-syntax
  3. Går det igenom kör deploy rsync över SSH
  4. Caddy på VPS:en serverar det nya — HTTPS via Tailscale hem
